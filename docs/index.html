Introduction

Welcome to the CITeR Projects Page documentation. This static website showcases research projects from the Center for Identification Technology Research.

This documentation will guide you through understanding the project structure, deploying the site, and contributing to the project.

Getting Started

To get started with this project, follow these steps:

  1. Clone the repository: git clone https://github.com/CITeR-Research/CITeR-Projects-Page.git
  2. Open index.html in your web browser
  3. Explore the projects and documentation

Requirements

  • A modern web browser (Chrome, Firefox, Safari, Edge)
  • A local web server (optional, for testing)

Project Structure

The project is organized as follows:

CITeR-Projects-Page/
├── index.html          # Main homepage
├── styles.css          # Main stylesheet
├── script.js           # JavaScript functionality
├── docs/               # Documentation folder
│   ├── index.html      # Documentation homepage
│   └── docs-style.css  # Documentation-specific styles
└── README.md           # Project readme

Deployment

This static website can be deployed to various platforms:

GitHub Pages

  1. Go to your repository settings
  2. Navigate to "Pages" in the sidebar
  3. Select the branch you want to deploy (e.g., main)
  4. Choose the root folder as the source
  5. Click "Save" and your site will be deployed

Netlify

  1. Connect your GitHub repository to Netlify
  2. Set the build command to empty (no build required)
  3. Set the publish directory to the root folder
  4. Deploy!

Vercel

  1. Import your GitHub repository to Vercel
  2. No build configuration needed
  3. Deploy automatically

API Reference

This static website doesn't require an API, but you can extend it with external APIs if needed.

JavaScript Functions

The script.js file includes the following functionality:

  • Smooth Scrolling: Automatically applied to all anchor links
  • Active Navigation: Updates nav links based on scroll position
  • Scroll Animations: Animates project cards on scroll

Contributing

We welcome contributions to the CITeR Projects Page! Here's how you can contribute:

  1. Fork the repository
  2. Create a new branch for your feature
  3. Make your changes
  4. Test your changes locally
  5. Submit a pull request

Coding Guidelines

  • Write clean, readable HTML
  • Follow CSS best practices
  • Keep JavaScript minimal and efficient
  • Ensure responsive design works on all devices
  • Test in multiple browsers
